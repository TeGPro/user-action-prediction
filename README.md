# User Action Prediction

Учебный проект по бинарной классификации: предсказание того, совершит ли пользователь целевое действие в веб-сессии.

Проект включает подготовку сессионных данных, feature engineering, сравнение нескольких моделей `scikit-learn` по ROC-AUC и HTTP API для инференса на FastAPI.

> Статус: учебный проект. Перед использованием результатов в реальной задаче необходимо устранить ограничения, перечисленные в разделе [Ограничения](#ограничения-и-дальнейшие-улучшения).

## Стек

- Python
- pandas
- scikit-learn
- FastAPI
- Uvicorn
- dill

## Что реализовано

- Загрузка и объединение данных веб-сессий и событий.
- Очистка пропусков и отбор признаков.
- Feature engineering: тип устройства, признак крупного города и премиального бренда.
- Обработка числовых и категориальных признаков через `ColumnTransformer`.
- Сравнение `LogisticRegression`, `RandomForestClassifier` и `MLPClassifier` с ROC-AUC cross-validation.
- Сохранение лучшего пайплайна и метаданных модели.
- FastAPI API с эндпоинтами `/predict` и `/version`.

## ML pipeline

1. Загружаются таблицы сессий и событий.
2. Сессионные данные объединяются с событиями по `session_id`.
3. Создается целевая переменная на основе заданного списка `event_action`.
4. Применяется очистка данных и feature engineering.
5. Числовые признаки проходят imputation и scaling; категориальные - imputation и one-hot encoding.
6. Несколько классификаторов сравниваются по ROC-AUC с cross-validation.
7. Лучший pipeline обучается на доступных данных и сохраняется для инференса.

## Запуск API

### 1. Установить зависимости

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install fastapi "uvicorn[standard]" pandas scikit-learn dill
```

### 2. Проверить артефакты модели

Для запуска API в каталоге `data/models/` должны находиться:

```text
ga_pipeline_final.pkl
custom_preprocessor.pkl
```

### 3. Запустить приложение

```bash
uvicorn main:app --reload
```

После запуска интерактивная документация доступна по адресу:

```text
http://127.0.0.1:8000/docs
```

## API

### `GET /version`

Возвращает метаданные сохраненной модели: версию, тип модели, дату обучения и ROC-AUC.

### `POST /predict`

Принимает данные одной веб-сессии в JSON и возвращает бинарное предсказание.

```json
{
  "session_id": "example-session-id",
  "device_brand": "Apple",
  "geo_city": "Moscow"
}
```

Пример ответа:

```json
{
  "session_id": "example-session-id",
  "prediction": 1
}
```

Полный набор входных полей зависит от признаков, использованных при обучении сохраненного pipeline. Перед интеграцией следует сверить контракт входных данных с метаданными модели и добавить Pydantic-схему запроса.

## Ограничения и дальнейшие улучшения

- Исходные CSV-файлы не включены в репозиторий; для полного воспроизведения обучения нужны данные и документированная инструкция их получения.
- `requirements.txt` или `pyproject.toml` отсутствует, поэтому версии зависимостей не зафиксированы.
- У API нет Pydantic-схемы, валидации входных полей и тестов.
- Логи, конфигурация, Docker и CI пока не настроены.
- Части feature engineering вычисляются до cross-validation. Для честной оценки все операции, зависящие от данных, включая отбор частотных категорий, нужно перенести внутрь folds.
- Пути, по которым training script сохраняет артефакты и FastAPI их загружает, следует унифицировать.
